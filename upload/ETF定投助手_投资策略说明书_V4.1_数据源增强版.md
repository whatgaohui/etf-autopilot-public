# ETF定投助手投资策略说明书 V4

> 文档定位：本文件是「投资决策规则」的权威说明。  
> 适用对象：AI开发系统、规则引擎开发、产品迭代、后续投资复盘。  
> 与 PRD 的关系：PRD 只描述产品需求、页面、接口、数据结构和验收标准；具体买入、暂停、再平衡、现金水池规则，以本策略文档为准。  
> 核心原则：**规则引擎算账，AI只解释；不预测涨跌，只维护纪律。**

---

## 1. 策略总纲

本系统不是短线择时工具，也不是普通ETF数据看板，而是一个：

```text
可审计、可解释、纪律化的个人ETF定投与再平衡决策系统
```

系统每周输出的不只是“买入清单”，而是一张：

```text
资金调拨执行单
```

它回答四个问题：

1. 本周新增资金买什么？
2. 哪些标的暂停买入？
3. 哪些标的需要做再平衡？
4. 未使用资金和再平衡释放资金去哪？

核心投资原则：

```text
低估多买
合理正常买
高估少买
极高估不买
极高估且超配才卖
卖也只卖超配部分
美股少卖
A股可周期再平衡
红利用股息率辅助判断
QDII高溢价禁止买入，但可辅助卖出
现金统一回到华宝添益
```

## 3. 估值周期选择规则

### 3.1 总体原则

估值分位不能只看一个周期。系统必须同时计算并展示：

```text
近1年分位
近3年分位
近5年分位
近10年分位 / 可得最长历史分位
```

但不同周期的用途不同：

| 周期 | 作用 | 是否参与强规则 |
|---|---|---|
| 近1年 | 短期情绪温度计 | 不参与强买卖规则 |
| 近3年 | 中期冷热辅助 | 辅助买入判断 |
| 近5年 | 默认主判断周期 | 参与买入、暂停、减量规则 |
| 近10年 / 全历史 | 大周期极值判断 | 参与再平衡规则 |
| 样本不足 | 标记数据不足 | 不触发强规则 |

核心原则：

```text
买入可以稍微灵敏，用5年为主；
卖出必须更保守，用10年或全历史为主；
近1年只做情绪参考，不直接触发买卖。
```

### 3.2 样本天数规则

| 样本天数 | 系统处理 |
|---:|---|
| >= 1000个交易日 | 可以参与近5年分位规则 |
| 500 ~ 999个交易日 | 可以展示，但标记“样本不足，仅供参考” |
| < 500个交易日 | 不参与强规则，只展示“数据不足” |
| 当前值缺失 | 不参与规则，进入数据质量异常 |

### 3.3 各类ETF估值周期口径

#### A股宽基：中证A500、沪深300、科创50

| 标的 | 买入规则周期 | 再平衡规则周期 | 特别说明 |
|---|---|---|---|
| 沪深300ETF | 5年PE/PB分位为主 | 10年或最长历史为主 | 历史较长，可做完整周期判断 |
| 科创50ETF | 5年/成立以来分位 | 成立以来 + 更宽阈值 | 高弹性成长，PE容易失真，需结合PB |
| 中证A500ETF | 自身分位展示，强规则谨慎 | 暂不建议强再平衡 | 指数历史短，应用沪深300/中证800/全A作为辅助代理 |

中证A500特别规则：

```text
如果中证A500自身历史样本不足5年：
- 展示自身估值分位，但标记“样本不足”
- 不直接触发强卖出规则
- 可用沪深300、中证800、全A宽基估值作为代理参考
```

#### 红利ETF

红利ETF重点看：

```text
股息率历史分位
```

解释：

```text
股息率高分位 = 相对便宜
股息率低分位 = 相对昂贵
```

推荐周期：

```text
股息率分位：5年为主，10年为辅
PE/PB：辅助参考
```

#### 美股宽基：标普500、纳斯达克

推荐周期：

```text
买入侧：5年 + 10年PE分位
卖出侧：10年 / 更长周期为主
```

美股允许估值高位钝化，因此：

```text
PE高可以暂停新增
但不能轻易卖核心仓
```

#### QDII溢价

QDII溢价不是长期估值指标，而是场内价格相对净值的短期偏离。

需要展示：

```text
当日溢价率
近3日均值
近7日均值
近20日/30日趋势
```

规则：

```text
买入否决：当日溢价 > 3%，且近3日/7日均值不低
卖出辅助：溢价 > 5%，且标的明显超配
```

## 4. 数据源与数据准确性策略（V4.1 增强版）

### 4.1 数据源策略总原则

数据源是本系统的“地基”。系统必须从“能抓到数据”升级为“知道数据从哪里来、准不准、能不能参与规则”。

本系统采用：

```text
多源接入 + 主备切换 + 交叉校验 + 数据质量评分 + 数据血缘留痕 + 异常熔断
```

关键原则：

```text
数据不可信，宁可不出建议；
数据可信，规则引擎才允许算账。
```

规则引擎不得直接使用外部接口返回值，必须使用经过以下流程处理后的数据：

```text
外部源 raw_data
→ 统一适配 Adapter
→ 原始数据入库
→ 清洗 clean_value
→ 主备源交叉校验
→ 数据质量评分
→ can_use_for_rule 判断
→ 规则引擎读取
```

关键字段必须至少支持“主源 + 备源 + 校验源”三层设计：

```text
ETF场内价格
基金净值
QDII溢价率
指数PE/PB
估值分位
红利股息率
ETF基础信息
交易日历
```

#### 4.3.3 QDII溢价率

QDII溢价率必须作为“衍生指标”计算，不应盲信某个网页展示字段。

计算公式：

```text
QDII溢价率 = 场内价格 / 最新单位净值 - 1
```

字段：

```text
market_price
unit_nav
nav_date
price_trade_date
premium_rate
premium_rate_3d_avg
premium_rate_7d_avg
premium_rate_20d_avg
premium_rate_30d_avg
nav_staleness_days
```

特殊规则：

```text
QDII净值存在时区与披露滞后。
买入否决不能只看单日溢价，至少结合当日 + 3日均值 / 7日均值。
卖出辅助可参考高溢价，但必须叠加“明显超配”。
```

建议规则：

```text
当日溢价 > 3% 且 3日均值 > 2.5%：买入一票否决
当日溢价 > 5% 且 当前占比超过目标5个百分点以上：可触发情绪溢价再平衡观察
```

### 4.6 数据源降级与熔断规则

当主源失败：

```text
尝试备源
备源成功且质量通过 → 使用备源，并标记 source_fallback=true
备源失败 → 使用最近有效缓存，并标记 stale=true
缓存过期 → 阻断自动建议
```

当主备源冲突：

```text
不盲目选择任一来源
标记 source_conflict
阻断强买入/强再平衡规则
提示人工确认
```

当只有单一数据源：

```text
允许展示
允许生成观察建议
不允许触发强再平衡
买入侧仅允许保守建议，并展示 single_source_warning
```

### 4.7 数据质量评分

每个指标生成0~100分数据质量评分：

```text
数据新鲜度：25分
主备源一致性：30分
字段完整性：20分
异常值检测：15分
数据源健康：10分
```

评级：

| 分数 | 状态 | 是否参与规则 |
|---:|---|---|
| >=90 | 优秀 | 可以参与强规则 |
| 75~89 | 可用 | 可以参与规则，但需标注 |
| 60~74 | 可疑 | 默认不参与强规则，仅展示 |
| <60 | 不可用 | 不参与规则 |

规则：

```text
买入建议至少要求数据质量 >=75
再平衡建议至少要求数据质量 >=90
QDII溢价买入否决至少要求价格和净值数据均可用
```

## 5. 数据质量与ETL清洗规则（V4.1 增强版）

### 5.1 原始数据与清洗数据分离

数据库必须至少分为四层：

```text
market_data_raw：原始数据
market_data_clean：清洗数据
source_compare_result：主备源比较结果
data_quality_result：数据质量评分结果
```

规则引擎只能读取：

```text
market_data_clean.clean_value
```

### 5.2 缺失值与异常值原则

规则：

```text
缺失值用 None / null 表示
禁止使用 99999999、-1、0 作为缺失值占位
禁止异常值进入图表
禁止异常值进入规则引擎
```

统一清洗函数：

```python
def clean_numeric(value):
    if value is None:
        return None

    try:
        x = float(value)
    except Exception:
        return None

    if math.isnan(x) or math.isinf(x):
        return None

    if abs(x) >= 999999:
        return None

    return x
```

## 6. 买入侧策略：目标缺口驱动

### 6.1 核心公式

```text
当前定投资产总额 = 6只定投ETF当前市值之和

预算后定投资产总额 = 当前定投资产总额 + 本周新增预算

某ETF预算后目标市值 = 预算后定投资产总额 × 该ETF目标占比

某ETF目标缺口 = 预算后目标市值 - 当前实际持仓市值

基础可买金额 = max(0, 目标缺口)
```

### 6.2 买入侧规则优先级

```text
一票否决 > 数据质量控制 > 减量 > 加量 > 目标缺口分配
```

减量和加量同时命中：

```text
减量优先
```

### 6.3 一票否决规则

| 规则 | 条件 | 结果 |
|---|---|---|
| QDII溢价红线 | 标普500ETF / 纳斯达克ETF 溢价率 > 3% | 本周不买 |
| 估值极高分位 | PE分位 > 80% | 本周不买 |
| 黑名单 | 标的在黑名单中 | 永不参与定投 |
| 关键数据不足 | 数据缺失、异常、样本不足 | 默认不自动买入，需人工确认 |

注意：

```text
A500样本不足时，PE分位不应直接触发强卖出规则；
但买入端可用代理估值和数据质量提示做保守处理。
```

### 6.4 减量规则

| 规则 | 条件 | 结果 |
|---|---|---|
| QDII溢价预警 | 溢价率 2%~3% | 买入金额 × 0.5 |
| 估值偏高 | PE分位 60%~80% | 买入金额 × 0.5 |
| 持仓过度集中 | 当前占比 > 目标占比 × 1.5 | 买入金额 × 0.5 或不买 |

### 6.5 加量规则

| 规则 | 条件 | 结果 |
|---|---|---|
| 估值极度低估 | PE/PB分位 < 20% | 买入金额 × 1.5~2 |
| 严重欠配 | 当前占比 < 目标占比 × 0.5 | 买入金额 × 1.5~2 |
| 红利极具吸引力 | 股息率分位 > 80% | 红利ETF可加量 |

## 7. 再平衡策略：退出机制

### 7.1 再平衡不是清仓择时

系统中不使用“卖出信号”这个概念，统一称为：

```text
再平衡建议
```

再平衡不是预测顶部，而是当资产同时满足：

```text
极高估 + 明显超配
```

时，卖出超配部分，维护组合纪律。

核心原则：

```text
卖的是超配部分，不卖核心底仓
```

超额市值公式：

```text
超额市值 = 当前持仓市值 - 当前定投资产总额 × 目标占比
```

### 7.2 A股宽基再平衡规则

适用标的：

```text
中证A500ETF
沪深300ETF
科创50ETF
```

| 触发条件 | 系统动作 | 资金去向 |
|---|---|---|
| PE/PB分位 > 90%，且当前占比超过目标占比5个百分点以上 | 卖出超额市值的30% | 华宝添益 |
| PE/PB分位 > 95%，且当前占比超过目标占比8个百分点以上 | 卖出超额市值的50% | 华宝添益 |
| PE/PB极高但没有超配 | 不卖，只暂停新增 | 无 |
| 超配但估值不高 | 不卖，后续用新增资金自然稀释 | 无 |

### 7.3 红利ETF再平衡规则

红利ETF主指标：

```text
股息率历史分位
```

| 条件 | 动作 |
|---|---|
| 股息率分位 < 15%，且当前占比超过目标占比5个百分点以上 | 卖出超额市值的30% |
| 股息率分位 < 10%，且当前占比超过目标占比8个百分点以上 | 卖出超额市值的50% |
| 股息率低但没有超配 | 不卖，只暂停或减量买入 |

### 7.4 美股宽基再平衡规则

适用标的：

```text
标普500ETF
纳斯达克ETF
```

| 触发条件 | 系统动作 | 资金去向 |
|---|---|---|
| PE分位 > 95%，且当前占比超过目标占比10个百分点以上 | 极端再平衡：卖出超额市值的20% | 华宝添益 |
| QDII溢价率 > 5%，且当前占比超过目标占比5个百分点以上 | 情绪溢价再平衡：卖出超额市值的30%~50% | 华宝添益 |
| 仅PE分位 > 90%，但没有超配 | 不卖，只暂停新增 | 无 |
| 仅QDII溢价高，但没有超配 | 不卖，只禁止新增 | 无 |

关键原则：

```text
QDII高溢价对买入是坏事；
如果本来满足再平衡条件，高溢价可作为更好的卖出窗口。
```

## 8. 现金水池规则

现金水池标的：

```text
华宝添益ETF 511990
```

资金流入来源：

```text
本周未分配新增预算
再平衡卖出释放资金
数据异常时暂不投入的资金
```

默认规则：

```text
所有未分配和再平衡释放资金先进入华宝添益
不默认二次分配到其他资产
```

现金水池风险：

```text
如果规则过于保守，现金比例可能长期偏高，产生现金拖累。
```

建议系统增加提示：

| 条件 | 提示 |
|---|---|
| 华宝添益占总资产 > 20% | 提醒现金占比偏高，建议复核阈值 |
| 华宝添益占总资产 > 30% | 提醒可能存在长期现金拖累 |
| 有极低估且严重欠配资产 | 可提示是否手动动用现金水池 |

## 10. 潜在风险与防护策略

### 10.1 策略风险

| 风险 | 表现 | 防护 |
|---|---|---|
| PE失真 | 成长/周期指数盈利波动导致PE误判 | 加入PB、股息率、盈利趋势 |
| 高估值钝化 | 美股长期处于高估值仍继续上涨 | 美股卖出更克制 |
| 规则过拟合 | 历史回测好，未来失效 | 阈值采用区间，不做过度优化 |
| 现金拖累 | 长期不买导致现金比例过高 | 设置现金水池提示阈值 |
| A股美股相关性上升 | 极端风险下一起下跌 | 监控组合最大回撤和权益仓位 |

### 10.2 数据风险

| 风险 | 表现 | 防护 |
|---|---|---|
| A500历史样本不足 | 分位失真 | 标记样本不足，使用代理指数 |
| QDII净值滞后 | 溢价率被高估或低估 | 看当日、3日、7日均值 |
| 估值口径不一致 | 不同平台PE/PB差异大 | 固定口径，展示数据源 |
| 数据源失效 | API字段变化 | 主备源切换，失败告警 |
| 异常值污染图表 | 出现99999999等异常 | ETL清洗 + 前端过滤 |

### 10.3 系统风险

| 风险 | 表现 | 防护 |
|---|---|---|
| OCR识别错误 | 持仓市值错误，目标缺口错误 | 人工校准 + 合计校验 |
| 前后端金额不一致 | 页面金额和规则引擎不同 | 前端只展示final_amount |
| LLM改金额 | AI文案篡改计算结果 | AI输出一致性校验 |
| 缓存过期 | 使用旧数据生成建议 | 数据新鲜度提示 |
| 缺少回溯 | 无法解释历史建议 | calculation_log记录 |

## 12. 最终策略总结

一句话总结：

```text
用目标缺口算法决定买入，
用估值与超配双条件决定再平衡，
用多周期估值避免短期噪音，
用多源数据校验保障准确性，
用华宝添益承接未投资和止盈资金，
用AI解释但不干预金额。
```

最终目标：

```text
不无脑死拿
不频繁择时
不清仓赌博
不情绪交易
用规则维护长期资产配置纪律
```
