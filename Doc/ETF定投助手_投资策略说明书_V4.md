# ETF定投助手投资策略说明书 V4

> 文档定位：本文件是「投资决策规则」的权威说明。  
> 适用对象：AI开发系统、规则引擎开发、产品迭代、后续投资复盘。  
> 与 PRD 的关系：PRD 只描述产品需求、页面、接口、数据结构和验收标准；具体买入、暂停、再平衡、现金水池规则，以本策略文档为准。  
> 核心原则：**规则引擎算账，AI只解释；不预测涨跌，只维护纪律。**

## 1. 策略总纲

本系统不是短线择时工具，也不是普通ETF数据看板，而是一个：

```text
可审计、可解释、纪律化的个人ETF定投与再平衡决策系统
```

系统每周输出的不只是“买入清单”，而是一张：

```text
资金调拨执行单
```

它回答四个问题：

1. 本周新增资金买什么？
2. 哪些标的暂停买入？
3. 哪些标的需要做再平衡？
4. 未使用资金和再平衡释放资金去哪？

核心投资原则：

```text
低估多买
合理正常买
高估少买
极高估不买
极高估且超配才卖
卖也只卖超配部分
美股少卖
A股可周期再平衡
红利用股息率辅助判断
QDII高溢价禁止买入，但可辅助卖出
现金统一回到华宝添益
```

## 3. 估值周期选择规则

### 3.1 总体原则

估值分位不能只看一个周期。系统必须同时计算并展示：

```text
近1年分位
近3年分位
近5年分位
近10年分位 / 可得最长历史分位
```

但不同周期的用途不同：

| 周期 | 作用 | 是否参与强规则 |
|---|---|---|
| 近1年 | 短期情绪温度计 | 不参与强买卖规则 |
| 近3年 | 中期冷热辅助 | 辅助买入判断 |
| 近5年 | 默认主判断周期 | 参与买入、暂停、减量规则 |
| 近10年 / 全历史 | 大周期极值判断 | 参与再平衡规则 |
| 样本不足 | 标记数据不足 | 不触发强规则 |

核心原则：

```text
买入可以稍微灵敏，用5年为主；
卖出必须更保守，用10年或全历史为主；
近1年只做情绪参考，不直接触发买卖。
```

### 3.2 样本天数规则

| 样本天数 | 系统处理 |
|---:|---|
| >= 1000个交易日 | 可以参与近5年分位规则 |
| 500 ~ 999个交易日 | 可以展示，但标记“样本不足，仅供参考” |
| < 500个交易日 | 不参与强规则，只展示“数据不足” |
| 当前值缺失 | 不参与规则，进入数据质量异常 |

### 3.3 各类ETF估值周期口径

#### A股宽基：中证A500、沪深300、科创50

| 标的 | 买入规则周期 | 再平衡规则周期 | 特别说明 |
|---|---|---|---|
| 沪深300ETF | 5年PE/PB分位为主 | 10年或最长历史为主 | 历史较长，可做完整周期判断 |
| 科创50ETF | 5年/成立以来分位 | 成立以来 + 更宽阈值 | 高弹性成长，PE容易失真，需结合PB |
| 中证A500ETF | 自身分位展示，强规则谨慎 | 暂不建议强再平衡 | 指数历史短，应用沪深300/中证800/全A作为辅助代理 |

中证A500特别规则：

```text
如果中证A500自身历史样本不足5年：
- 展示自身估值分位，但标记“样本不足”
- 不直接触发强卖出规则
- 可用沪深300、中证800、全A宽基估值作为代理参考
```

#### 红利ETF

红利ETF重点看：

```text
股息率历史分位
```

解释：

```text
股息率高分位 = 相对便宜
股息率低分位 = 相对昂贵
```

推荐周期：

```text
股息率分位：5年为主，10年为辅
PE/PB：辅助参考
```

#### 美股宽基：标普500、纳斯达克

推荐周期：

```text
买入侧：5年 + 10年PE分位
卖出侧：10年 / 更长周期为主
```

美股允许估值高位钝化，因此：

```text
PE高可以暂停新增
但不能轻易卖核心仓
```

#### QDII溢价

QDII溢价不是长期估值指标，而是场内价格相对净值的短期偏离。

需要展示：

```text
当日溢价率
近3日均值
近7日均值
近20日/30日趋势
```

规则：

```text
买入否决：当日溢价 > 3%，且近3日/7日均值不低
卖出辅助：溢价 > 5%，且标的明显超配
```

## 4. 数据源与数据准确性策略

### 4.1 数据源总原则

本系统不要只依赖单一数据源，必须采用：

```text
主源 + 备源 + 交叉校验 + 数据质量评分
```

关键字段必须至少两个来源交叉验证：

```text
ETF收盘价
基金净值
QDII溢价率
PE分位
PB分位
股息率
```

### 4.2 推荐数据源分层

#### 免费 / 低成本层

| 数据源 | 推荐角色 | 适用数据 | 注意事项 |
|---|---|---|---|
| AkShare | 快速开发源、备源 | 基金、ETF、指数、估值、净值等 | 免费易用，但部分接口依赖网页，不能作为唯一权威源 |
| Tushare Pro | 主源之一 | 指数行情、基金行情、基金净值、指数指标 | 需要积分/权限，接口稳定性较适合程序化 |
| JQData / 聚宽 | 主源或校验源 | 股票、基金、指数、历史行情、量化数据 | 需要账号与权限，适合量化研究 |

#### 专业商业层

| 数据源 | 推荐角色 | 适用数据 | 注意事项 |
|---|---|---|---|
| Wind | 终极权威源/机构级校验 | 全市场行情、基金、指数、宏观、估值 | 成本较高 |
| Choice | 商业校验源 | 基金、指数、估值、宏观 | 需授权 |
| iFinD | 商业校验源 | 金融数据、估值、行情 | 需授权 |
| 指数公司官网 | 权威校验源 | 指数估值、指数股息率、指数样本 | 接口化能力需确认 |
| 基金公司官网 | 权威校验源 | 基金净值、公告、分红 | 适合关键净值校验 |

### 4.3 推荐数据源架构

```text
行情价格：
主源：Tushare fund_daily / JQData
备源：AkShare / 东方财富
校验：交易所行情或券商行情

基金净值：
主源：Tushare fund_nav / JQData extras
备源：AkShare / 天天基金
校验：基金公司官网披露净值

QDII溢价：
计算公式：场内收盘价 / 最新单位净值 - 1
主源：场内价格 + 基金净值
备源：东方财富/天天基金折溢价字段
校验：基金公司净值更新时间

指数估值：
主源：中证指数 / 国证指数 / Wind / JQData / Tushare
备源：AkShare指数估值
校验：第三方估值平台只做参考，不做唯一来源

红利股息率：
主源：指数公司 / Wind / JQData
备源：AkShare / 东方财富
校验：指数官网或基金定期报告
```

### 4.4 双源校验容忍范围

| 字段 | 容忍差异 |
|---|---:|
| ETF收盘价 | 0.1%以内 |
| 基金净值 | 0.3%以内 |
| QDII溢价率 | 0.5个百分点以内 |
| PE/PB原始值 | 2%以内 |
| PE/PB分位 | 3~5个百分点以内 |
| 股息率 | 0.2个百分点以内 |

超过阈值时：

```text
不自动生成强买卖建议
页面提示：数据源不一致，建议人工确认
```

### 4.5 数据血缘字段

每个指标必须记录：

```text
source
source_api
raw_value
clean_value
updated_at
trade_date
fetch_time
is_valid
abnormal_reason
sample_days
percentile_window
```

数据库不应只存最终值，必须保留原始值与清洗值。

## 5. 数据质量与ETL清洗规则

### 5.1 缺失值与异常值原则

规则：

```text
缺失值用 None / null 表示
禁止使用 99999999、-1、0 作为缺失值占位
```

所有外部数据进入缓存前，必须统一清洗：

```python
def clean_numeric_series(s):
    s = pd.to_numeric(s, errors="coerce")
    s = s.replace([np.inf, -np.inf], np.nan)
    s = s.mask(s.abs() >= 999999)
    return s
```

### 5.2 异常值规则

| 指标 | 异常过滤建议 |
|---|---|
| PE | <= 0 或 >= 500 视为异常，除非业务确认 |
| PB | <= 0 或 >= 100 视为异常 |
| 溢价率 | abs(溢价率) > 30% 标记异常 |
| 股息率 | < 0 或 > 20% 标记异常 |
| 净值 | <= 0 视为异常 |
| 市值 | <= 0 视为异常 |

异常值处理：

```text
raw_value保留原始值
clean_value置为null
is_valid=false
记录abnormal_reason
```

### 5.3 数据质量状态

| 状态 | 含义 | 是否参与规则 |
|---|---|---|
| 通过 | 双源一致，数据新鲜，样本充足 | 是 |
| 轻微异常 | 小幅差异，但在容忍范围内 | 可参与，需标注 |
| 严重异常 | 差异超阈值或关键字段异常 | 否 |
| 数据不足 | 样本不足或字段缺失 | 否 |
| 源不一致 | 主备源差异超阈值 | 否 |

### 5.4 缓存过期规则

| 数据类型 | 过期规则 |
|---|---|
| A股ETF收盘价 | 非最新交易日，标红 |
| QDII净值 | 超过2个交易日未更新，标黄 |
| 指数估值 | 超过2个交易日未更新，标黄 |
| 溢价率 | 当日缺失，标红 |
| 股息率 | 超过7个交易日未更新，标黄 |

## 6. 买入侧策略：目标缺口驱动

### 6.1 核心公式

```text
当前定投资产总额 = 6只定投ETF当前市值之和

预算后定投资产总额 = 当前定投资产总额 + 本周新增预算

某ETF预算后目标市值 = 预算后定投资产总额 × 该ETF目标占比

某ETF目标缺口 = 预算后目标市值 - 当前实际持仓市值

基础可买金额 = max(0, 目标缺口)
```

### 6.2 买入侧规则优先级

```text
一票否决 > 数据质量控制 > 减量 > 加量 > 目标缺口分配
```

减量和加量同时命中：

```text
减量优先
```

### 6.3 一票否决规则

| 规则 | 条件 | 结果 |
|---|---|---|
| QDII溢价红线 | 标普500ETF / 纳斯达克ETF 溢价率 > 3% | 本周不买 |
| 估值极高分位 | PE分位 > 80% | 本周不买 |
| 黑名单 | 标的在黑名单中 | 永不参与定投 |
| 关键数据不足 | 数据缺失、异常、样本不足 | 默认不自动买入，需人工确认 |

注意：

```text
A500样本不足时，PE分位不应直接触发强卖出规则；
但买入端可用代理估值和数据质量提示做保守处理。
```

### 6.4 减量规则

| 规则 | 条件 | 结果 |
|---|---|---|
| QDII溢价预警 | 溢价率 2%~3% | 买入金额 × 0.5 |
| 估值偏高 | PE分位 60%~80% | 买入金额 × 0.5 |
| 持仓过度集中 | 当前占比 > 目标占比 × 1.5 | 买入金额 × 0.5 或不买 |

### 6.5 加量规则

| 规则 | 条件 | 结果 |
|---|---|---|
| 估值极度低估 | PE/PB分位 < 20% | 买入金额 × 1.5~2 |
| 严重欠配 | 当前占比 < 目标占比 × 0.5 | 买入金额 × 1.5~2 |
| 红利极具吸引力 | 股息率分位 > 80% | 红利ETF可加量 |

## 7. 再平衡策略：退出机制

### 7.1 再平衡不是清仓择时

系统中不使用“卖出信号”这个概念，统一称为：

```text
再平衡建议
```

再平衡不是预测顶部，而是当资产同时满足：

```text
极高估 + 明显超配
```

时，卖出超配部分，维护组合纪律。

核心原则：

```text
卖的是超配部分，不卖核心底仓
```

超额市值公式：

```text
超额市值 = 当前持仓市值 - 当前定投资产总额 × 目标占比
```

### 7.2 A股宽基再平衡规则

适用标的：

```text
中证A500ETF
沪深300ETF
科创50ETF
```

| 触发条件 | 系统动作 | 资金去向 |
|---|---|---|
| PE/PB分位 > 90%，且当前占比超过目标占比5个百分点以上 | 卖出超额市值的30% | 华宝添益 |
| PE/PB分位 > 95%，且当前占比超过目标占比8个百分点以上 | 卖出超额市值的50% | 华宝添益 |
| PE/PB极高但没有超配 | 不卖，只暂停新增 | 无 |
| 超配但估值不高 | 不卖，后续用新增资金自然稀释 | 无 |

### 7.3 红利ETF再平衡规则

红利ETF主指标：

```text
股息率历史分位
```

| 条件 | 动作 |
|---|---|
| 股息率分位 < 15%，且当前占比超过目标占比5个百分点以上 | 卖出超额市值的30% |
| 股息率分位 < 10%，且当前占比超过目标占比8个百分点以上 | 卖出超额市值的50% |
| 股息率低但没有超配 | 不卖，只暂停或减量买入 |

### 7.4 美股宽基再平衡规则

适用标的：

```text
标普500ETF
纳斯达克ETF
```

| 触发条件 | 系统动作 | 资金去向 |
|---|---|---|
| PE分位 > 95%，且当前占比超过目标占比10个百分点以上 | 极端再平衡：卖出超额市值的20% | 华宝添益 |
| QDII溢价率 > 5%，且当前占比超过目标占比5个百分点以上 | 情绪溢价再平衡：卖出超额市值的30%~50% | 华宝添益 |
| 仅PE分位 > 90%，但没有超配 | 不卖，只暂停新增 | 无 |
| 仅QDII溢价高，但没有超配 | 不卖，只禁止新增 | 无 |

关键原则：

```text
QDII高溢价对买入是坏事；
如果本来满足再平衡条件，高溢价可作为更好的卖出窗口。
```

## 8. 现金水池规则

现金水池标的：

```text
华宝添益ETF 511990
```

资金流入来源：

```text
本周未分配新增预算
再平衡卖出释放资金
数据异常时暂不投入的资金
```

默认规则：

```text
所有未分配和再平衡释放资金先进入华宝添益
不默认二次分配到其他资产
```

现金水池风险：

```text
如果规则过于保守，现金比例可能长期偏高，产生现金拖累。
```

建议系统增加提示：

| 条件 | 提示 |
|---|---|
| 华宝添益占总资产 > 20% | 提醒现金占比偏高，建议复核阈值 |
| 华宝添益占总资产 > 30% | 提醒可能存在长期现金拖累 |
| 有极低估且严重欠配资产 | 可提示是否手动动用现金水池 |

未投资金额：
- 0 元

## 10. 潜在风险与防护策略

### 10.1 策略风险

| 风险 | 表现 | 防护 |
|---|---|---|
| PE失真 | 成长/周期指数盈利波动导致PE误判 | 加入PB、股息率、盈利趋势 |
| 高估值钝化 | 美股长期处于高估值仍继续上涨 | 美股卖出更克制 |
| 规则过拟合 | 历史回测好，未来失效 | 阈值采用区间，不做过度优化 |
| 现金拖累 | 长期不买导致现金比例过高 | 设置现金水池提示阈值 |
| A股美股相关性上升 | 极端风险下一起下跌 | 监控组合最大回撤和权益仓位 |

### 10.2 数据风险

| 风险 | 表现 | 防护 |
|---|---|---|
| A500历史样本不足 | 分位失真 | 标记样本不足，使用代理指数 |
| QDII净值滞后 | 溢价率被高估或低估 | 看当日、3日、7日均值 |
| 估值口径不一致 | 不同平台PE/PB差异大 | 固定口径，展示数据源 |
| 数据源失效 | API字段变化 | 主备源切换，失败告警 |
| 异常值污染图表 | 出现99999999等异常 | ETL清洗 + 前端过滤 |

### 10.3 系统风险

| 风险 | 表现 | 防护 |
|---|---|---|
| OCR识别错误 | 持仓市值错误，目标缺口错误 | 人工校准 + 合计校验 |
| 前后端金额不一致 | 页面金额和规则引擎不同 | 前端只展示final_amount |
| LLM改金额 | AI文案篡改计算结果 | AI输出一致性校验 |
| 缓存过期 | 使用旧数据生成建议 | 数据新鲜度提示 |
| 缺少回溯 | 无法解释历史建议 | calculation_log记录 |

## 12. 最终策略总结

一句话总结：

```text
用目标缺口算法决定买入，
用估值与超配双条件决定再平衡，
用多周期估值避免短期噪音，
用多源数据校验保障准确性，
用华宝添益承接未投资和止盈资金，
用AI解释但不干预金额。
```

最终目标：

```text
不无脑死拿
不频繁择时
不清仓赌博
不情绪交易
用规则维护长期资产配置纪律
```
