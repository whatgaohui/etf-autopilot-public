# PRD v4.1 新增章节：数据源管理模块

> 可直接插入 PRD v4，建议作为第10章，原第10章及后续章节顺延。

## 10. 数据源管理模块需求

### 10.1 模块定位

数据源管理模块用于管理：

```text
数据从哪里来
数据准不准
数据是否新鲜
数据是否能参与规则计算
数据异常时系统如何降级
```

模块目标：

```text
从单一AkShare抓取
升级为
多源接入、主备校验、质量评分、异常熔断、可回溯数据血缘
```

### 10.2 核心能力

1. 数据源注册中心 DataSource Registry。
2. 指标级主源/备源/校验源配置。
3. 数据源适配器 DataSource Adapter。
4. 数据获取调度 Data Fetch Scheduler。
5. 数据清洗引擎 Data Cleaning Engine。
6. 主备源交叉校验 Cross Source Validator。
7. 数据质量评分 Data Quality Score。
8. 数据降级与熔断机制。
9. 设置页数据源管理页面。
10. 总览页数据可信度卡。
11. 趋势页指标数据血缘展示。

### 10.3 数据源注册字段

```text
source_id
source_name
source_type
provider_type
base_url
auth_type
api_key_required
rate_limit
enabled
priority
supported_metrics
last_success_time
last_error_time
health_status
error_message
```

### 10.4 指标级数据源配置

| 指标 | 主源 | 备源 | 校验源 |
|---|---|---|---|
| ETF场内价格 | Tushare / JQData | AkShare / efinance | 东方财富 / 交易所 |
| 基金净值 | Tushare / JQData | AkShare / 天天基金 | 基金公司官网 |
| QDII溢价率 | 自算：价格/净值-1 | 东方财富展示值 | 净值更新时间校验 |
| 指数估值 | 指数公司 / Tushare / JQData | AkShare | Wind / Choice / iFinD |
| 红利股息率 | 指数公司 / 商业源 | AkShare / 东方财富 | 指数官网 / 基金报告 |

### 10.5 数据质量评分

```text
数据新鲜度：25分
主备源一致性：30分
字段完整性：20分
异常值检测：15分
数据源健康：10分
```

| 分数 | 状态 | 是否参与规则 |
|---:|---|---|
| >=90 | 优秀 | 可以参与强规则 |
| 75~89 | 可用 | 可以参与规则，但需标注 |
| 60~74 | 可疑 | 默认不参与强规则，仅展示 |
| <60 | 不可用 | 不参与规则 |

### 10.6 数据降级规则

```text
主源失败 → 尝试备源
备源成功 → 使用备源并标记source_fallback=true
备源失败 → 使用最近有效缓存并标记stale=true
缓存过期 → 阻断自动建议
主备冲突 → 标记source_conflict并阻断强规则
只有单源 → 允许展示，限制强再平衡
```

### 10.7 页面要求

设置页新增「数据源管理」Tab，包含：

1. 数据源列表。
2. 指标级主备源配置。
3. API Token配置。
4. 数据源启用/停用。
5. 数据源连接测试。
6. 最近拉取状态。
7. 错误日志。
8. 主备源差异阈值配置。
9. 手动刷新数据。
10. 数据质量结果查看。
11. 数据血缘查看。
12. 数据源异常处理提示。

# PRD v4.2 新增章节：宏观环境模块（极简宏观温度计）

> 可直接插入 PRD v4.1，建议作为「数据源管理模块」之后的独立章节。  
> 模块定位：宏观环境模块不是交易决策引擎，不直接计算买入/卖出金额；它只作为「风险提示、背景解释、复盘归因」使用。  
> 核心原则：**宏观看方向，估值定贵贱，缺口算金额，溢价控买点，数据质量决定能不能出建议。**

## 11. 宏观环境模块需求

### 11.1 模块定位

宏观环境模块用于回答：

```text
当前宏观环境对哪些资产是顺风？
当前宏观环境对哪些资产是逆风？
本周是否出现了需要提示用户的宏观异常？
这些宏观异常如何解释本周的市场波动？
```

但宏观模块不得回答：

```text
本周具体买多少钱？
本周是否卖出？
是否清仓？
是否覆盖规则引擎金额？
```

系统权责边界必须固定：

| 模块 | 权限 |
|---|---|
| 规则引擎 | 计算买入金额、再平衡金额、现金水池流向 |
| 数据质量引擎 | 判断数据是否可用于规则 |
| 宏观温度计 | 生成客观风险提示 |
| AI解释层 | 基于规则引擎和宏观提示做解释 |
| 前端 | 展示结果，不二次计算金额 |

---

### 11.2 宏观模块双层架构

宏观模块分为两层：

```text
执行层：极简宏观温度计
研究层：低频宏观研究面板
```

#### A. 执行层

用于每周生成执行单时展示。

特点：

```text
日频更新
市场化定价
客观指标
只在异常时提示
不直接影响金额
```

执行层只保留 4 个核心日频指标：

| 维度 | 指标 | 作用 |
|---|---|---|
| 国内无风险利率 | 中国10年期国债收益率 | 国内资产定价锚、红利ETF相对债券吸引力 |
| 全球利率锚 | 美国10年期国债收益率 | 全球折现率、纳指/标普/科创50成长股压力 |
| 汇率与外资情绪 | USD/CNH 离岸人民币汇率 | 外资情绪、人民币压力、QDII人民币收益影响 |
| 极端风险温度 | VIX恐慌指数 | 全球风险偏好和极端抛售信号 |

#### B. 研究层

用于趋势页深度分析、月度复盘、季度复盘。

特点：

```text
月频 / 季频
低频发布
更适合解释长期环境
不进入每周执行单主流程
不直接触发买入或再平衡
```

研究层指标包括：

```text
社融、M1/M2、PMI、CPI/PPI、工业企业利润、社零、固定资产投资、地产销售/投资、
非农、失业率、核心PCE、ISM PMI、高收益债利差、信用利差、商品价格
```

---

### 11.3 为什么执行层不接入大量月频宏观指标

系统周频定投，每周二生成执行单；而社融、PMI、CPI、PPI、工业利润等指标大多是月频或季频。

如果将这些低频指标放入每周执行层，会导致：

```text
频次错配
数据滞后
指标长期不变
每周提示重复
与估值买入规则产生认知冲突
```

因此：

```text
月频宏观指标不进入每周执行单核心区域
月频宏观指标不参与金额计算
月频宏观指标只进入研究层和复盘层
```

### 11.4 极简宏观温度计指标设计

#### 11.4.1 中国10年期国债收益率

作用：

```text
国内无风险利率锚
红利ETF股息率相对吸引力锚
A股估值折现率参考
```

影响逻辑：

```text
中国10年国债收益率下行
→ 无风险收益下降
→ 高股息资产相对吸引力增强
→ 红利ETF更有支撑

中国10年国债收益率快速上行
→ 无风险收益上升
→ 股息资产相对吸引力下降
→ 高估值资产折现压力上升
```

系统展示示例：

```text
本周中国10年国债收益率快速上行，红利资产相对债券的吸引力边际下降。本提示不改变买入金额，仅作为风险说明。
```

不允许：

```text
因中国10年国债收益率变化直接卖出红利ETF
因中国10年国债收益率变化覆盖目标缺口算法
```

---

#### 11.4.2 美国10年期国债收益率

作用：

```text
全球资产定价锚
海外成长股估值压力指标
纳斯达克、标普500、科创50波动提示指标
```

影响逻辑：

```text
美国10年期国债收益率快速上行
→ 全球折现率上升
→ 高估值成长股承压
→ 纳斯达克、标普500、科创50短期波动风险上升

美国10年期国债收益率快速下行
→ 折现率压力缓解
→ 成长资产估值环境改善
```

系统展示示例：

```text
本周美国10年期国债收益率快速上行，海外成长股短期波动风险加大。系统仍按目标缺口、估值分位和QDII溢价规则计算买入金额。
```

不允许：

```text
因美国10年期国债收益率上行直接暂停所有美股买入
因美国10年期国债收益率上行直接触发卖出
```

---

#### 11.4.3 USD/CNH 离岸人民币汇率

作用：

```text
人民币汇率压力
外资风险偏好
全球美元流动性
QDII人民币计价收益解释
```

影响逻辑：

```text
USD/CNH上行
→ 人民币贬值
→ 外资风险偏好下降
→ A股宽基可能承压
→ 美股QDII人民币计价收益可能被汇率抬高

USD/CNH下行
→ 人民币升值
→ 外资风险偏好可能改善
→ A股估值环境改善
→ 美股QDII人民币计价收益可能被汇率抵消
```

系统展示示例：

```text
本周离岸人民币明显走弱，A股宽基短期可能承压；若系统触发低估买入，建议继续保持分批纪律。
```

不允许：

```text
因人民币贬值直接停止A股低估买入
因人民币升值直接加仓A股
```

---

#### 11.4.4 VIX 恐慌指数

作用：

```text
全球极端风险温度计
风险偏好剧烈变化提示
现金水池战略价值提示
```

影响逻辑：

```text
VIX快速上升
→ 市场隐含波动率上升
→ 全球风险资产抛售风险增加

VIX突破高位阈值
→ 全球系统性风险或恐慌情绪升温
→ 现金水池的战略价值上升
```

系统展示示例：

```text
本周VIX快速升至高位，全球风险偏好明显下降。本提示不触发卖出，只提示市场可能进入高波动阶段。
```

不允许：

```text
因VIX升高直接清仓
因VIX升高直接暂停所有买入
```

### 11.5 宏观提示触发规则

宏观指标只在出现剧烈变化时提示。

默认触发阈值：

| 指标 | 触发条件 | 提示级别 |
|---|---|---|
| 中国10年期国债收益率 | 单周变化 >= 15bp | 普通提示 |
| 中国10年期国债收益率 | 单周变化 >= 25bp | 强提示 |
| 美国10年期国债收益率 | 单周变化 >= 25bp | 普通提示 |
| 美国10年期国债收益率 | 单周变化 >= 35bp | 强提示 |
| USD/CNH | 单周变化 >= 1% | 普通提示 |
| USD/CNH | 单周变化 >= 2% | 强提示 |
| VIX | 突破 30 | 强提示 |
| VIX | 单周上升 >= 30% | 普通提示 |
| VIX | 单周上升 >= 50% | 强提示 |

说明：

```text
阈值为初始建议，后续可根据历史回测和用户体验调整。
```

### 11.6 宏观提示展示原则

前端不展示复杂、主观的宏观大标签，例如：

```text
Risk-on
Risk-off
宽信用
紧信用
衰退交易
复苏交易
```

原因：

```text
这类标签容易造成主观解释
容易和规则引擎结论冲突
容易让用户重新陷入人工纠结
```

前端只展示客观提示句。

示例：

```text
宏观提示：
本周美国10年期国债收益率上行32bp，海外成长股短期波动风险加大。本提示不改变买入金额。
```

```text
宏观提示：
本周VIX突破30，全球风险偏好明显下降。系统不会因此自动卖出，仅提示高波动环境。
```

如果无明显异常：

```text
宏观温度计：本周无明显宏观异常。
```

### 11.7 宏观模块与规则引擎边界

宏观模块只能做：

```text
解释
提示
标注
复盘归因
```

宏观模块不能做：

```text
不能修改买入金额
不能触发卖出
不能触发再平衡
不能覆盖估值规则
不能覆盖QDII溢价规则
不能覆盖目标缺口算法
不能让AI根据宏观主观改金额
```

### 11.9 宏观数据质量要求

宏观指标同样必须进入数据质量体系。

字段：

```text
metric_type
raw_value
clean_value
source_id
trade_date
fetch_time
quality_score
quality_status
can_use_for_macro_prompt
abnormal_reason
```

规则：

```text
quality_score < 60：不展示宏观提示
数据过期：不展示宏观提示
主备源严重冲突：不展示宏观提示
单源可用：允许展示，但标记single_source_warning
```

注意：

```text
宏观数据不可用时，不影响规则引擎生成买入/再平衡建议。
```

原因：

```text
宏观模块是解释层，不是执行层。
```

### 11.11 总览页宏观提示

总览页只在异常时展示宏观提示。

位置：

```text

```

展示形式：

```text
一行或两行客观备注
```


- [ ] 宏观指标有数据质量状态。
- [ ] 宏观提示可回溯。

#### 边界验收

| 场景 | 期望 |
|---|---|
| 美国10年期国债收益率单周上行35bp | 展示海外成长风险提示，但不改金额 |
| VIX突破30 | 展示高波动提示，但不触发卖出 |
| USD/CNH单周上行2% | 展示人民币压力提示，但不暂停A股买入 |
| 中国10年国债收益率单周上行25bp | 展示红利/估值锚提示，但不卖红利ETF |
| 宏观数据缺失 | 不展示宏观提示，不影响规则引擎 |
| 宏观数据源冲突 | 不展示宏观提示，提示数据异常 |
| 估值规则建议买入，但宏观提示偏谨慎 | 仍按规则引擎金额展示，只附提示 |
| 宏观提示与AI解释冲突 | AI解释必须服从规则引擎金额 |

## 13. 最终模块原则

一句话原则：

```text
宏观看方向，估值定贵贱，缺口算金额，溢价控买点，数据质量决定能不能出建议。
```

系统最终权责边界：

```text
规则引擎负责执行
宏观模块负责提示
AI负责解释
用户负责确认
```
