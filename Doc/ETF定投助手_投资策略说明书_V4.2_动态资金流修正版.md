# ETF定投助手投资策略说明书 V4.2

> 文档定位：本文件是「投资决策规则」的权威说明。  
> 适用对象：AI开发系统、规则引擎开发、产品迭代、后续投资复盘。  
> 对应产品文档：PRD v4.2（数据源管理 + 宏观温度计模块）。  
> 与 PRD 的关系：PRD 描述产品需求、页面、接口、数据库和验收标准；本策略文档描述投资决策逻辑、资金流转规则、买入/暂停/再平衡/现金水池规则。  
> 核心原则：**规则引擎算账，AI只解释；不预测涨跌，只维护纪律。**

---

## 0. V4.2 相比 V4.1 的关键更新

本版本在 V4.1 基础上重点修复 5 类问题：

1. **修复分母陷阱**：再平衡卖出进入华宝添益的钱，如果仍属于权益备用金，必须计入权益配置基准，避免卖出后整体目标盘子被动缩水。
2. **修复 QDII 溢价资金黑洞**：因 QDII 高溢价暂时买不出去的预算，不得无限堆积为一次性巨额买入，必须进入 QDII 挂起池，并按释放计划分批恢复。
3. **修复一票否决过硬问题**：把规则拆成“硬否决”和“软风控”。PE/PB 高估不再默认一票否决，而是限制增强仓、降低买入强度或仅允许基础定投。
4. **新增基础定投仓 + 估值增强仓**：每周预算拆分，保证定投底盘不因软风控完全停摆。
5. **新增宏观温度计边界**：宏观模块只做风险提示和解释，不修改金额、不触发卖出、不覆盖规则引擎。

本版本最终原则：

```text
宏观看方向
估值定贵贱
缺口算金额
溢价控买点
数据质量决定能不能出建议
现金水池记录资金身份
```

## 1. 策略总纲

本系统不是短线择时工具，也不是普通 ETF 数据看板，而是一个：

```text
可审计、可解释、纪律化的个人ETF定投与再平衡资金调拨系统
```

系统每周输出的不只是“买入清单”，而是一张：

```text
资金调拨执行单
```

它回答：

1. 本周新增资金买什么？
2. 哪些标的暂停买入？
3. 哪些标的只允许基础定投，不允许增强买入？
4. 哪些标的需要再平衡？
5. 未使用资金、再平衡资金、QDII挂起资金分别去哪？
6. 数据是否可信？
7. 宏观环境是否有异常提示？

核心投资原则：

```text
低估多买
合理正常买
高估少买
极高估限制买
极高估且超配才卖
卖也只卖超配部分
美股少卖
A股可周期再平衡
红利用股息率和债券利差判断
QDII高溢价硬刹车
现金水池必须区分资金身份
```

## 3. 资金池与资金身份

### 3.1 华宝添益不是单一现金池

华宝添益必须拆成多个虚拟子账户，不同资金身份对应不同策略含义。

| 子账户 | 含义 | 是否计入权益配置基准 |
|---|---|---|
| daily_cash | 日常现金，不打算投权益 | 否 |
| weekly_unallocated_cash | 本周未分配但仍计划未来投向权益的钱 | 是，可设置过期规则 |
| rebalance_equity_reserve | 再平衡卖出后暂存、等待未来重新配置的钱 | **是** |
| qdii_pending_cash_sp500 | 因标普500ETF QDII溢价过高暂缓买入的钱 | 是，并归属标普500目标 |
| qdii_pending_cash_nasdaq | 因纳斯达克ETF QDII溢价过高暂缓买入的钱 | 是，并归属纳斯达克目标 |
| manual_cash | 用户手动指定不参与本系统的钱 | 否 |

核心原则：

```text
日常现金 ≠ 权益备用金 ≠ QDII挂起资金
```

### 3.2 权益配置基准

V4.1 的旧公式：

```text
当前定投资产总额 = 6只定投ETF当前市值之和
```

在动态系统中存在分母缩水问题。V4.2 修正为：

```text
权益配置基准 =
6只定投ETF当前市值之和
+ rebalance_equity_reserve
+ weekly_unallocated_cash中仍计划投向权益的部分
+ qdii_pending_cash_sp500
+ qdii_pending_cash_nasdaq
```

不计入：

```text
daily_cash
manual_cash
黄金ETF华安
其他非本策略资产
```

### 3.3 目标市值计算

```text
某ETF目标市值 = 权益配置基准 × 该ETF目标占比
```

### 3.4 战略缺口与本周可执行缺口

系统必须区分两个概念：

```text
战略缺口 ≠ 本周可执行买入金额
```

#### 战略缺口

用于判断长期目标配置偏离。

```text
战略缺口 = 目标市值 - 当前ETF市值 - 已归属该ETF的挂起资金
```

其中，标普500和纳斯达克的 QDII 挂起资金要归属到对应 ETF，避免缺口无限滚大。

#### 本周可执行缺口

用于本周下单金额计算。

```text
本周可执行缺口 = min(战略缺口, 本周可用预算上限, 规则允许买入上限)
```

原则：

```text
缺口可以很大，但本周不能一把梭。
```

## 4. 每周预算分桶

### 4.1 预算拆分

每周新增预算分成：

| 资金桶 | 默认比例 | 作用 |
|---|---:|---|
| 基础定投仓 base_bucket | 40% | 保持长期定投纪律 |
| 估值增强仓 value_bucket | 60% | 根据估值、缺口、溢价做动态加减 |
| 现金水池 cash_bucket | 非固定 | 没有合适标的时承接未使用资金 |

示例：

```text
每周预算 7,000 元：
基础定投仓 = 2,800 元
估值增强仓 = 4,200 元
```

可配置参数：

```text
base_bucket_ratio 默认 40%
value_bucket_ratio 默认 60%
base_bucket_ratio 可在 30%~50% 之间调整
```

### 4.2 基础定投仓规则

基础定投仓只避开硬否决，不受普通 PE/PB 高估直接归零影响。

基础定投仓优先投向：

```text
无硬否决
相对欠配
估值风险较低
数据可用
```

当所有标的都触发软风控时：

```text
仍允许基础定投仓投向“最欠配 + 无硬否决”的标的
```

当所有标的都触发硬否决时：

```text
基础定投仓进入华宝添益，并记录为 weekly_unallocated_cash
```

### 4.3 估值增强仓规则

估值增强仓受估值、溢价、股息率、目标缺口严格约束。

当标的低估且欠配：

```text
增强仓优先分配
```

当标的偏高：

```text
增强仓减量或禁用
```

当增强仓无合适标的：

```text
进入华宝添益，记录为 weekly_unallocated_cash
```

## 5. 硬否决与软风控

### 5.1 硬否决

硬否决是真正的一票否决。

| 规则 | 条件 | 结果 |
|---|---|---|
| 黑名单 | 标的在黑名单中 | 永不参与定投 |
| 数据严重异常 | 关键数据缺失、过期、冲突、质量不可用 | 不自动买入/不自动再平衡 |
| QDII溢价红线 | 标普500/纳斯达克 溢价率 > 3%，且3日均值 > 2.5% | 不买该QDII |
| 停牌/流动性异常 | 无法正常成交 | 不买/不卖 |
| 净值/价格严重冲突 | 无法可信计算溢价 | 不买QDII |

硬否决效果：

```text
基础定投仓和估值增强仓都不得买入该标的
```

### 5.2 软风控

软风控不等于一票否决，主要影响增强仓。

| 规则 | 条件 | 结果 |
|---|---|---|
| PE/PB偏高 | 60%~80% | 增强仓减量 |
| PE/PB高估 | 80%~90% | 禁止增强仓，仅允许基础仓 |
| PE/PB极高 | 90%~95% | 仅严重欠配时允许极小额基础仓 |
| PE/PB极端 | >95% | 暂停新增，除非策略手动允许 |
| 宏观压力 | 宏观温度计异常 | 只提示，不改金额 |
| 轻微超配 | 当前占比略高于目标 | 减少或暂停增强仓 |
| 单源数据 | 只有单一数据源可用 | 可展示/可保守买入，但不触发强再平衡 |

核心原则：

```text
PE/PB高估禁止加速买入，不必然禁止所有买入。
```

## 6. 估值周期选择规则

### 6.1 总体原则

系统同时计算并展示：

```text
近1年分位
近3年分位
近5年分位
近10年分位 / 可得最长历史分位
```

用途：

| 周期 | 作用 | 是否参与强规则 |
|---|---|---|
| 近1年 | 短期情绪温度计 | 不参与强买卖规则 |
| 近3年 | 中期冷热辅助 | 辅助买入判断 |
| 近5年 | 默认主判断周期 | 参与买入、暂停、减量规则 |
| 近10年 / 全历史 | 大周期极值判断 | 参与再平衡规则 |
| 样本不足 | 标记数据不足 | 不触发强规则 |

核心原则：

```text
买入可以稍微灵敏，用5年为主；
卖出必须更保守，用10年或全历史为主；
近1年只做情绪参考，不直接触发买卖。
```

### 6.2 样本天数规则

| 样本天数 | 系统处理 |
|---:|---|
| >=1000个交易日 | 可以参与近5年分位规则 |
| 500~999个交易日 | 可以展示，但标记“样本不足，仅供参考” |
| <500个交易日 | 不参与强规则，只展示“数据不足” |
| 当前值缺失 | 不参与规则，进入数据质量异常 |

### 6.3 中证A500代理估值

中证A500自身历史短，不能直接用短历史分位触发强规则。

代理估值优先级：

| 优先级 | 代理指数 |
|---:|---|
| 1 | 中证800 |
| 2 | 中证全指 / 万得全A |
| 3 | 沪深300 |
| 4 | 不触发强估值规则，仅展示样本不足 |

规则：

```text
A500自身历史样本不足5年时：
买入侧估值参考优先使用中证800；
若中证800不可用，使用中证全指/万得全A；
若仍不可用，使用沪深300作为保守代理；
若代理估值也不可用，则不触发强估值规则，仅做数据不足提示。
```

### 6.4 红利ETF股息率反向逻辑

红利ETF重点看：

```text
股息率历史分位
股息率 - 中国10年国债收益率利差
```

防呆规则：

```text
PE/PB分位：越高越贵
股息率分位：越高越便宜
股息率分位：越低越贵
```

规则引擎必须为指标配置方向字段：

| 指标 | metric_direction | 含义 |
|---|---|---|
| PE分位 | higher_is_expensive | 越高越贵 |
| PB分位 | higher_is_expensive | 越高越贵 |
| QDII溢价率 | higher_is_expensive | 越高越不适合买 |
| 股息率分位 | higher_is_cheaper | 越高越便宜 |
| 股息率-10年国债利差 | higher_is_cheaper | 越高越有吸引力 |

红利ETF买入逻辑：

| 股息率分位 | 操作 |
|---:|---|
| >80% | 可加量 |
| 40%~80% | 正常 |
| 20%~40% | 减量 |
| <20% | 暂停增强买入 |
| <10% 且明显超配 | 可触发再平衡观察 |

## 7. 买入侧策略：目标缺口驱动 V4.2

### 7.1 买入算法步骤

```text
1. 读取持仓快照和现金子账户
2. 读取清洗后的市场数据
3. 判断数据是否可用
4. 计算权益配置基准
5. 计算各ETF目标市值
6. 计算战略缺口
7. 识别硬否决标的
8. 拆分基础定投仓和估值增强仓
9. 基础仓投向无硬否决且相对欠配标的
10. 增强仓按估值、缺口、溢价规则分配
11. 无法分配的预算进入对应现金子账户
12. 生成资金调拨执行单
```

### 7.2 目标缺口公式

```text
权益配置基准 =
6只ETF当前市值
+ rebalance_equity_reserve
+ weekly_unallocated_cash中仍计划投向权益的部分
+ qdii_pending_cash_sp500
+ qdii_pending_cash_nasdaq
```

```text
某ETF目标市值 = 权益配置基准 × 目标占比
```

```text
某ETF战略缺口 =
目标市值
- 当前ETF市值
- 已归属该ETF的挂起资金
```

```text
基础可买金额 = max(0, 战略缺口)
```

### 7.3 本周可执行金额封顶

```text
本周可执行金额 =
min(
    战略缺口,
    本周预算可分配金额,
    标的单周买入上限,
    QDII挂起释放上限,
    规则允许金额
)
```

建议默认单只ETF单周买入上限：

```text
max_single_etf_buy_ratio = 70% × 本周新增预算
```

对于 QDII 挂起资金释放：

```text
单周释放金额不得超过该ETF正常周预算的 1.5~2倍
```

## 8. QDII溢价挂起池规则

### 8.1 触发条件

适用：

```text
标普500ETF 513500
纳斯达克ETF 513300
```

当：

```text
当日溢价率 > 3%
且 3日均值 > 2.5%
```

则触发 QDII 硬否决。

### 8.2 资金流向

因 QDII 高溢价无法买入的预算，不进入普通现金池，而进入对应挂起池：

```text
标普500被阻断 → qdii_pending_cash_sp500
纳斯达克被阻断 → qdii_pending_cash_nasdaq
```

### 8.3 缺口防滚雪球

QDII挂起资金在战略缺口计算中视为已归属该 ETF 的待投资额度。

```text
标普500战略缺口 =
标普500目标市值
- 标普500当前市值
- qdii_pending_cash_sp500
```

这样可以避免因长期溢价阻断导致战略缺口无限滚大。

### 8.4 释放规则

当 QDII 溢价恢复正常：

```text
当日溢价率 <= 3%
且 3日均值 <= 2.5%
```

挂起资金不得一次性释放，必须分批释放。

默认释放计划：

```text
release_plan_weeks = 8周
单周释放上限 = min(挂起余额 / 剩余释放周数, 该ETF正常周预算 × 2)
```

可配置范围：

```text
release_plan_weeks: 4~12周
release_cap_multiplier: 1.5~2.0
```

### 8.5 长期溢价锁死提示

如果同一 QDII 连续 4 周因溢价被阻断：

```text
系统提示：场内QDII溢价持续偏高，建议人工评估是否使用场外联接基金/场外QDII方式承接对应额度。
```

系统不得自动切换场外基金。

原因：

```text
场外产品涉及申赎费率、确认日、限购、跟踪误差、汇率口径和流动性，必须人工确认。
```

## 9. 再平衡策略：退出机制 V4.2

### 9.1 再平衡不是清仓择时

系统不使用“卖出信号”这个概念，统一称为：

```text
再平衡建议
```

再平衡不是预测顶部，而是当资产同时满足：

```text
极高估 + 明显超配
```

时，卖出超配部分，维护组合纪律。

核心原则：

```text
卖的是超配部分，不卖核心底仓。
```

### 9.2 再平衡分母修正

旧公式：

```text
超额市值 = 当前持仓市值 - 当前定投资产总额 × 目标占比
```

V4.2 修正为：

```text
超额市值 = 当前持仓市值 - 权益配置基准 × 目标占比
```

其中，权益配置基准必须包含：

```text
6只ETF市值
+ rebalance_equity_reserve
+ 仍计划投向权益的weekly_unallocated_cash
+ QDII挂起资金
```

### 9.3 再平衡资金流向

再平衡卖出的资金默认进入：

```text
rebalance_equity_reserve
```

而不是普通 daily_cash。

该资金仍计入权益配置基准，避免分母缩水。

只有用户手动确认“永久降低权益仓位”时，才可从：

```text
rebalance_equity_reserve → daily_cash
```

### 9.4 A股宽基再平衡规则

适用标的：

```text
中证A500ETF
沪深300ETF
科创50ETF
```

| 触发条件 | 系统动作 | 资金去向 |
|---|---|---|
| PE/PB分位 >90%，且当前占比超过目标5个百分点以上 | 卖出超额市值的30% | rebalance_equity_reserve |
| PE/PB分位 >95%，且当前占比超过目标8个百分点以上 | 卖出超额市值的50% | rebalance_equity_reserve |
| PE/PB极高但没有超配 | 不卖，只暂停增强买入 | 无 |
| 超配但估值不高 | 不卖，后续用新增资金自然稀释 | 无 |

### 9.5 红利ETF再平衡规则

红利ETF主指标：

```text
股息率历史分位
股息率 - 中国10年国债收益率利差
```

| 条件 | 动作 |
|---|---|
| 股息率分位 <15%，且当前占比超过目标5个百分点以上 | 卖出超额市值的30% |
| 股息率分位 <10%，且当前占比超过目标8个百分点以上 | 卖出超额市值的50% |
| 股息率低但没有超配 | 不卖，只暂停或减量买入 |

强调：

```text
红利ETF股息率分位越低，代表越贵；
不要把股息率分位逻辑写反。
```

### 9.6 美股宽基再平衡规则

适用标的：

```text
标普500ETF
纳斯达克ETF
```

| 触发条件 | 系统动作 | 资金去向 |
|---|---|---|
| PE分位 >95%，且当前占比超过目标10个百分点以上 | 极端再平衡：卖出超额市值的20% | rebalance_equity_reserve |
| QDII溢价率 >5%，且当前占比超过目标5个百分点以上 | 情绪溢价再平衡：卖出超额市值的30%~50% | rebalance_equity_reserve |
| 仅PE分位 >90%，但没有超配 | 不卖，只暂停增强买入 | 无 |
| 仅QDII溢价高，但没有超配 | 不卖，只禁止新增 | 无 |

## 10. 全体否决兜底机制

### 10.1 触发条件

当系统发现：

```text
6只核心ETF最终买入金额均为0
```

必须进入兜底判断，不能直接输出“本周不投”。

### 10.2 兜底判断

| 原因 | 系统动作 |
|---|---|
| 全部因数据异常 | 不生成自动建议，要求人工确认 |
| 全部因硬否决 | 本周预算进入华宝添益对应子账户 |
| 全部因软风控 | 启动基础定投仓 |
| QDII因溢价高，美股无法买 | 对应预算进入QDII挂起池 |
| A股估值偏高但无硬否决 | 基础仓投向最欠配且风险相对较低标的 |
| 所有标的严重高估 | 预算进入 weekly_unallocated_cash，并提示现金拖累风险 |

### 10.3 输出要求

执行单必须明确说明：

```text
本周不投是因为硬否决，还是软风控？
如果是软风控，是否已启动基础定投仓？
未投资金进入哪个现金子账户？
该资金未来是否仍计入权益配置基准？
```

宏观模块不能做：

不能覆盖QDII溢价规则

## 14. 潜在风险与防护策略

### 14.1 策略风险

| 风险 | 表现 | 防护 |
|---|---|---|
| 分母缩水 | 再平衡后权益目标被动下降 | 权益配置基准计入权益备用金 |
| QDII资金黑洞 | 高溢价长期阻断，后续一次性大额买入 | QDII挂起池 + 分批释放 |
| 全体软否决 | 每周都不投，定投失效 | 基础定投仓兜底 |
| PE失真 | 成长/周期指数盈利波动导致PE误判 | 加入PB、股息率、盈利趋势 |
| 高估值钝化 | 美股高估值仍继续上涨 | 美股卖出更克制 |
| 规则过拟合 | 历史回测好，未来失效 | 阈值采用区间，不做过度优化 |
| 现金拖累 | 长期现金比例过高 | 现金子账户 + 现金占比提示 |

### 14.2 数据风险

| 风险 | 表现 | 防护 |
|---|---|---|
| A500历史样本不足 | 分位失真 | 明确代理指数优先级 |
| QDII净值滞后 | 溢价率被误判 | 当日 + 3日/7日均值 |
| 股息率逻辑写反 | 红利ETF高位加仓 | metric_direction 防呆 |
| 数据源失效 | API字段变化 | 强缓存 + Fail Fast |
| 异常值污染图表 | 出现99999999等异常 | raw/clean拆分 + 清洗 |

### 14.3 系统风险

| 风险 | 表现 | 防护 |
|---|---|---|
| OCR识别错误 | 持仓市值错误 | 人工校准 + 合计校验 |
| 前后端金额不一致 | 展示金额和引擎金额不同 | 前端只展示 final_amount |
| LLM改金额 | AI文案篡改结果 | AI一致性校验 |
| 缺少资金身份 | 华宝添益资金含义混乱 | 现金子账户管理 |
| 缺少回溯 | 无法解释历史建议 | calculation_log + cash_ledger |

## 16. 最终策略总结

一句话总结：

```text
用权益配置基准稳定目标盘子，
用目标缺口算法决定战略方向，
用基础定投仓保持定投纪律，
用估值增强仓提高资金效率，
用硬否决避免真风险，
用软风控控制节奏，
用QDII挂起池解决高溢价资金黑洞，
用再平衡权益备用金修复分母缩水，
用宏观温度计解释环境但不改金额，
用AI解释但不干预金额。
```

最终目标：

```text
不无脑死拿
不频繁择时
不清仓赌博
不情绪交易
不因规则过硬导致长期停投
不因现金池设计错误导致权益仓位螺旋萎缩
用规则维护长期资产配置纪律
```
