## 0. 编制说明

本文档基于对 PRD v4.1 全 18 章节的逐条审阅，结合对当前代码库的**双视角审计**（Python data-service 后端 + Next.js 前端）输出。审计方法：
- 后端审计：扫描 `mini-services/data-service/` 全部 routers / services / models / scheduler，对照 PRD §10~§13 数据源管理 + §11 规则引擎 + §12 API + §13 数据库表。
- 前端审计：扫描 `src/components/` 全部组件，对照 PRD §7 总览页 + §8 趋势页 + §9 设置页 + §10.10 数据源管理页面。

**核心结论**：上一轮（DATA-SRC-1/2/3）已交付数据源管理模块的"骨架"——4 适配器元数据 + 字段级主备源配置 + 交叉校验日志 + 数据血缘查询 + 4 Tab UI。但 PRD v4.1 要求的"数据源治理模块"是一个**纵深防御体系**，目前只完成了"配置层"，缺失"质量评分层"、"原始/清洗数据分层存储层"、"拉取日志层"、"降级熔断层"、"前端可信度聚合卡"。

## 2. 迭代计划总览（5 个阶段）

按 PRD §17 的 P0/P1/P2/P3 优先级 + 缺失项严重度，拆成 5 个迭代阶段。每阶段独立可交付、可验收。

```
阶段 1（P0+）：数据源治理底座 —— 评分体系 + 原始/清洗分层 + 拉取日志
阶段 2（P0+）：多源冗余 —— efinance 免费备源 + AkShare 子源拆分 + 熔断降级链路
阶段 3（P1）：调度治理 + 响应体补全 —— 定时任务跑交叉校验 + CalculateResponse 补字段
阶段 4（P1）：总览页数据可信度卡 + 趋势页血缘接入 —— 前端可信度聚合
阶段 5（P2/P3）：设置页数据源管理深度完善 + 官方校验源适配器
```

### 阶段依赖关系

```
阶段1 ──→ 阶段3 ──→ 阶段4
  │                    ↑
  └──→ 阶段2 ──────────┘
                 阶段5（可并行，依赖 1+2）
```

### 3.2 验收标准

- [ ] `market_data_raw` 表存在，每次刷新有写入（SELECT count(*) > 0）
- [ ] `market_data_clean` 表存在，UNIQUE 约束生效（重复插入不报错，REPLACE 更新）
- [ ] `data_fetch_log` 表存在，每条拉取有 request_id 追踪
- [ ] `data_quality_result` 表存在，6 只 ETF × 5 指标 = 30 条质量记录
- [ ] 质量分公式单测：freshness=25（当日）/ consistency=30（主备一致）/ completeness=20（5 字段齐全）/ abnormal=15（无异常）/ source_health=10（源健康）= 100 分
- [ ] 规则引擎：当某 ETF 质量分 < 75 时，buy_suggestions 中该 ETF `vetoed=true` 且 reason 含"数据质量分"
- [ ] `GET /api/data-quality/summary` 返回 `{total_metrics, excellent, usable, suspicious, unavailable, avg_score}`
- [ ] lint 0 error

### 3.3 关键设计决策

**质量分公式细化**：
```
freshness(25) = 当日 25 / 1 日内 20 / 2 日内 15 / 3 日内 10 / >3 日 0
consistency(30) = 主备源 diff ≤ 通过阈值 30 / ≤ 严重阈值 15 / > 严重阈值 0 / 无备源 20(降级)
completeness(20) = 5 字段全有 20 / 缺 1 字段 15 / 缺 2 字段 10 / 缺 ≥3 字段 0
abnormal(15) = 无异常 15 / 有 1 异常 8 / 有 ≥2 异常 0
source_health(10) = 主源健康 10 / 主源降级 5 / 主源故障 0
```

**`market_data_raw` vs `market_data_clean` 分层意义**：
- raw 表保留**原始拉取快照**（含 raw_json 完整 payload），用于排障和回溯
- clean 表存**清洗后数值**（clean_value 已通过 safe_num + abnormal 检测），规则引擎只读 clean
- 两表通过 (code, trade_date, metric_type, source_id) 关联
